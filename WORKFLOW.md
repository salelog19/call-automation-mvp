# WORKFLOW

## Зачем нужен этот файл

Этот файл нужен, чтобы проект не "жил только в чате".
Если переписка в VS Code потеряется, сменится окно, проект или сессия, контекст можно будет восстановить по файлам в репозитории.

## Главное правило

Не считать чат постоянной памятью проекта.
Постоянная память проекта - это файлы в папке проекта.

## Как вести работу по проекту

1. Обсуждаем задачу в чате.
2. Все важные решения сразу фиксируем в файлах проекта.
3. После каждой сессии обновляем состояние проекта.
4. Перед переключением на другой проект сохраняем короткий итог.
5. При возвращении просим Codex восстановить контекст из файлов.

## Какие файлы за что отвечают

- `README.md` - что это за проект, цель, общий обзор
- `docs/mvp.md` - границы MVP, сценарии, что входит и не входит
- `docs/schema.md` - схема данных, сущности, связи
- `DECISIONS.md` - принятые решения и причины
- `TASKS.md` - текущие, следующие и завершенные задачи

## Минимальная привычка после каждой полезной сессии

В конце работы писать Codex одну из фраз:

- `Обнови документацию по итогам`
- `Сохрани решения в DECISIONS.md и следующие шаги в TASKS.md`
- `Зафиксируй текущее состояние проекта`

## Что делать перед переключением на другой проект

Перед уходом полезно зафиксировать:

- что уже сделано
- что делать следующим шагом
- какие есть открытые вопросы

Удобная команда для Codex:

`Сделай краткий status snapshot проекта и обнови DECISIONS.md / TASKS.md`

## Что делать при возвращении в проект

Если старая переписка пропала, написать:

`Восстанови контекст проекта по файлам и продолжим`

Если документы обновлялись регулярно, этого обычно достаточно, чтобы быстро продолжить работу.

## Практичный ритм работы

Хороший цикл для новичка:

1. Сформулировать идею или задачу.
2. Попросить Codex предложить следующий маленький шаг.
3. Сделать этот шаг.
4. Сразу сохранить решение и статус в файлах проекта.
5. Перейти к следующему шагу.

## Что не стоит делать

- не держать важные решения только в чате
- не надеяться, что VS Code всегда сохранит именно эту сессию
- не копить много несохраненных договоренностей
- не прыгать дальше в код, если не зафиксированы базовые решения

## Самая полезная рабочая привычка

В конце каждой заметной сессии просить:

`Обнови файлы проекта так, чтобы завтра можно было продолжить без истории чата`

Это и есть самый надежный способ вести проект вместе с Codex.
